서블릿(Servlet)


웹 서버 내부에 동작하는 작은 자바 프로그램이다.

클라이언트로부터 요청이 들어올 때 문자열 파싱과 검증과 같은 번거로운 부분을 처리하고 온전히 비즈니스 로직에 집중할 수 있도록 해준다.


위와 같이 HTTP 요청 정보를 Java 객체로 변환해주기 때문에 개발자는 이를 사용하여 편리하게 비즈니스 로직을 구현할 수 있다.

서블릿 컨테이너의 HTTP 요청/응답 과정



이와 같이 서블릿을 지원하는 Web Application Server를 "서블릿 컨테이너"라고 한다.


서블릿 컨테이너 특징

1. 서블릿 객체를 생성,초기화,호출,종료에 대한 라이프 사이클을 관리
2. 서블릿 객체는 싱글톤으로 관리
3. 동시 요청에 대한 멀티 쓰레드 지원

동시 요청 - 멀티 쓰레드

쓰레드란?

위에서 서블릿 객체를 호출해주는 역할을 하는 것이 쓰레드
각 요청을 처리하는 데에 하나의 쓰레드가 필요하다.

쓰레드를 하나만 사용할 경우 

여러 요청이 들어올 경우 대기해야 하는 상황이 발생할 수 있다.

요청마다 쓰레드를 생성하고 응답 완료 후 쓰레드를 소멸시킨다면

동시 요청에 대한 완벽한 처리가 가능하지만 쓰레드의 생성 비용이 높고 서버 임계점을 넘는 쓰레드를 생성할 경우 서버가 죽을 수 있다.

쓰레드 풀



WAS 내부에 일정 개수의 쓰레드를 미리 생성해두고, 요청이 올 때마다 해당 쓰레드 풀에서 쓰레드를 꺼내서 사용한다.
그리고 사용이 끝나면 다시 반납한다.(톰캣의 경우 default 값은 200)

 - 쓰레드를 미리 생성해두고 소멸시키지 않기 때문에 생성/소멸에 대한 비용이 절약되고 응답 시간이 빠르다.
 - 생성 가능한 쓰레드의 최대치가 정해져 있어 너무 많은 요청이 들어와도 안전하게 처리가 가능하다.
